# 祭坛状态物件设计（PropEntity 扩展为状态物件）

> **状态：设计评审中（未实施）。** 把 PropEntity 从"过场一次性角色"扩展为"可挡路 + 可遮挡 + scenario/flag 驱动切 clip 的状态物件"，首个用例是 prologue 祭坛（no_bowl ↔ with_bowl）。
> **关联**：[Prologue 收尾三项设计.MD](./Prologue%20收尾三项设计.MD)、[PROJECT_CONTEXT.md §6.13/6.14](../PROJECT_CONTEXT.md)。

## 0. 背景与目标

Prologue 有个任务：主人公捡道具交 Charlotte，Charlotte 复原祭坛。需求是祭坛交任务前后视觉变化（台面无钵 → 有钵）。

方案选择过程：
- NpcCharacter 方案：杀鸡用牛刀（强制带 NpcController/occupancy/interactables，stateGraph clip 切换无人驱动）
- 新建 SceneryEntity：与 PropEntity 高度重叠，过度设计
- **本方案：扩展现有 PropEntity**，加 blocker/depthMask/stateMap 三个能力，复用现有 actorRegistry/ExploreCollisionSystem 体系

未来同模式复用：宝箱（空/满）、门（关/开）、机关（未触发/已触发）、篝火（灭/燃）。一次性投入，长期收益。

## 1. 现状缺口

### 1.1 PropEntity 无占位能力
- [PropEntity.js](file:///e:/se/BlindDuel/scripts/Enties/PropEntity.js) 无 `getBlockerAabb()`
- [ExploreMode._buildIndices](file:///e:/se/BlindDuel/scripts/Systems/Modes/ExploreMode.js#L405-L449) staticBlockers 只收 `kind==="npc" && blocksMovement`，PropEntity 即使实现接口也不被收集
- 无 occupancy 扫描依赖（PropEntity root 来自 `config.pos` + 固定几何偏移，和 rabble 一致）

### 1.2 PropEntity 无遮挡能力
- 现有遮挡走 [SceneVisualSystem.createDepthMasks](file:///e:/se/BlindDuel/scripts/Enties/SceneVisualSystem.js#L34-L113)，按整张场景背景扫描，场景级静态、不随实体存在性变化
- STAGE 层 `parallaxFactor: 1.0`（[SceneVisualSystem.js:586](file:///e:/se/BlindDuel/scripts/Enties/SceneVisualSystem.js#L586)），`layerOffsetX = cameraAnchorX * (1 - 1.0) = 0`，本就不滚动；祭坛与 hero 同属 STAGE 层语义，钉在世界坐标，无背景视差移动

### 1.3 PropEntity 无状态驱动
- clip 切换靠 sequence SEND_COMMAND（手动），无 scenario/flag 自动响应
- NpcCharacter 的 stateGraph 也是手动 enterState，同样不自动响应 WorldState

### 1.4 Scene._onWorldStateChange 只处理 _pendingSpawns
- [Scene.js:451-461](file:///e:/se/BlindDuel/scripts/Scene.js#L451-L461) 当前只检查未生成的 entity，不处理已存在 entity 的状态切换

### 1.5 PropEntity 只支持多文件 clip，不支持单文件多 tag
- [PropEntity._setClip](file:///e:/se/BlindDuel/scripts/Enties/PropEntity.js#L58-L113) 每个 clip 独立 `spriteSheetUrl` + `atlasData`，切 clip 时 `material.diffuseTexture.dispose()` + `new Texture(...)`（[L95-L97](file:///e:/se/BlindDuel/scripts/Enties/PropEntity.js#L95-L97)），texture 异步加载会有闪烁
- [NpcFrameComponent](file:///e:/se/BlindDuel/scripts/Components/NpcFrameComponent.js#L40-L56) 已支持单文件多 tag（`atlas.meta.frameTags` + clip 的 `from/to`），texture 共享不换。PropEntity 未跟进
- 大尺寸 prop（祭坛/宝箱/门）多状态切文件会闪、占显存、美术拆文件工作流不友好；应复用 NPC 的 tag 机制

## 2. 设计要点

| 维度 | 决策 |
|---|---|
| 实体类型 | 复用 PropEntity，不新建类 |
| root 语义 | sprite 底边中点在地面投影（PropEntity 类硬编码，与 rabble 一致） |
| 碰撞 | `blocker` AABB 配置，进 ExploreCollisionSystem.staticBlockers |
| 遮挡 | `depthMask` AABB 配置，建 stencil plane 挂 root（与 sprite 同父；STAGE 层 parallaxFactor=1.0 不滚动，自然同步） |
| 钵 | 画进 with_bowl clip 的 sprite 内部，被主体包裹，无独立遮挡/sprite |
| clip 切换 | stateMap 不动 depthMask，只切 sprite texture |
| 状态驱动 | `stateMap: [{if, clip}]` 有序条件数组，复用 `_evaluateCondition`，支持 scenario/flag 混用 |
| 驱动时机 | Scene._onWorldStateChange + 首次 spawn 评估 |
| 初始 clip | stateMap 首个命中 > initialClip 兜底 > clips 首项 |
| 交互 | 无（祭坛纯视觉） |

### 2.1 root 锚点（与 rabble 一致）

[PropEntity.js:17](file:///e:/se/BlindDuel/scripts/Enties/PropEntity.js#L17) `this.root.position.set(pos[0], pos[1], pos[2])`，[PropEntity.js:53](file:///e:/se/BlindDuel/scripts/Enties/PropEntity.js#L53) `spritePlane.position.y = planeH / 2`。

→ root = sprite 底边中点在地面投影。`pos[1]=0` 让 sprite 底边贴地。不依赖扫描数据（PropEntity 全文不读 rootMotion/occupancy）。

### 2.2 blocker 字段（碰撞盒，相对 root 本地坐标）

```jsonc
"blocker": { "halfW": 1.2, "halfH": 0.9, "centerY": 0.9 }
```

- `centerY`：盒子几何中心相对 root 的 y（世界坐标）。台面贴地则 `centerY = halfH`；台面悬空则 `centerY = halfH + 悬空高度`
- 与 AABBTrigger.pos 语义一致（中心点）

PropEntity 实现：
```js
getBlockerAabb() {
    if (!this._blocker) return null;
    const p = this.root.position;
    const b = this._blocker;
    return {
        minX: p.x - b.halfW, maxX: p.x + b.halfW,
        minY: b.centerY - b.halfH, maxY: b.centerY + b.halfH
    };
}
```

### 2.3 depthMask 字段（遮挡盒，相对 root 本地坐标）

```jsonc
"depthMask": { "halfW": 1.3, "halfH": 2.0, "centerY": 1.0 }
```

字段结构与 blocker 完全一致，语义不同：
- blocker：碰撞盒（挡路），由 ExploreCollisionSystem 消费
- depthMask：遮挡盒（走背后变透明），由 stencil plane 实现

**关键设计**：depthMask plane 挂 `this.root`（与 spritePlane 同父）。STAGE 层 `parallaxFactor: 1.0`（[SceneVisualSystem.js:586](file:///e:/se/BlindDuel/scripts/Enties/SceneVisualSystem.js#L586)），`layerOffsetX` 恒为 0，本就不滚动——祭坛与 hero 同属 STAGE 层语义，钉在世界坐标，无背景视差移动：
- depthMask 与 sprite 同步移动（root 动两者一起动）
- STAGE 层不滚动，hero 走动时祭坛与 depthMask 都钉死，stencil 遮挡始终对齐 sprite

stencil 实现复用 SceneVisualSystem.createDepthMasks 的 `onBeforeRenderObservable`/`onAfterRenderObservable` 逻辑，抽成工具函数：

```js
// scripts/Utils/StencilOccluder.js（新增，或挂 SceneVisualSystem 静态方法）
export function attachStencilOccluder(plane, scene) {
    const gl = scene.getEngine()._gl;
    plane.onBeforeRenderObservable.add(() => {
        gl.colorMask(false, false, false, false);
        gl.enable(gl.STENCIL_TEST);
        gl.stencilMask(0xFF);
        gl.stencilFunc(gl.ALWAYS, 1, 0xFF);
        gl.stencilOp(gl.KEEP, gl.KEEP, gl.REPLACE);
    });
    plane.onAfterRenderObservable.add(() => {
        gl.colorMask(true, true, true, true);
        gl.stencilFunc(gl.NOTEQUAL, 1, 0xFF);
        gl.stencilOp(gl.KEEP, gl.KEEP, gl.KEEP);
        gl.stencilMask(0x00);
    });
}
```

PropEntity._buildSpritePlane 末尾，若 config.depthMask 存在则建一个 stencil plane 并 attach。

### 2.4 stateMap 字段（状态驱动 clip 切换）

- 多触发源混用：祭坛用 scenarioMin，宝箱用 flag，门用 flag 链

### 2.5 clip 模式约定（单文件多 tag vs 多文件）

PropEntity 支持两种 clip 配置方式，**祭坛等大尺寸状态物件优先用单文件多 tag**：

**方式 A：单文件多 tag（推荐，复用 [NpcFrameComponent](file:///e:/se/BlindDuel/scripts/Components/NpcFrameComponent.js#L40-L56) 机制）**
```jsonc
"spriteSheetUrl": "./Art/Sprite/altar/altar.png",
"atlasData": { /* meta.frameTags: [{name:"no_bowl",from:0,to:0},{name:"with_bowl",from:1,to:1}] */ },
"clips": {
  "no_bowl":   { "tag": "no_bowl",   "mode": "hold" },
  "with_bowl": { "tag": "with_bowl", "mode": "hold" }
}
```
- texture 在 entity 级建一次，切 clip 只改 `currentFrameIndex` + `_applyFrame`，**零 texture 切换、零闪烁**
- fixedUpdate 循环/hold 边界限制在 `tag.from ~ tag.to`
- 美术在 ase 同一文件画所有状态帧 + 设 frameTags，导出一份 png + json

**方式 B：多文件（旧式，rabble 保持不变，兼容）**
```jsonc
"clips": {
  "idle": { "spriteSheetUrl": "rabble_idle.png", "atlasData": {...} },
  "fall": { "spriteSheetUrl": "rabble_fall.png", "atlasData": {...} }
}
```
- clip 有 `spriteSheetUrl` 走旧逻辑（换 texture）
- 判断：clip 有 `tag` 字段 → 共享 entity 级 texture；clip 有 `spriteSheetUrl` → 独立 texture

**mode 约定**：静态状态（祭坛 no_bowl/with_bowl）用单帧 atlas + `mode:"hold"`。若未来要"钵渐变出现"效果，with_bowl clip 改为多帧动画 + `mode:"hold"`，播完自动定格末帧——无需新增"按帧停止"能力，现有 hold 模式即可。

### 2.6 钵的视觉约束（美术侧）

- with_bowl 帧 = 祭坛主体 + 钵（钵画在帧内部）
- no_bowl 帧 = 祭坛主体（钵不存在）
- 两帧在同一 atlas（单文件多 tag），外轮廓一致（都是主体轮廓），只是内部多了/少了钵
- 钵位置完全被祭坛主体 sprite 包裹 → 遮挡关系随主体，depthMask 只需一个 AABB（主体轮廓），clip 切换不动遮罩

## 3. 驱动时机

### 3.1 首次 spawn 评估

[Scene._spawnEntity](file:///e:/se/BlindDuel/scripts/Scene.js#L395-L428) 末尾，若 entity 有 stateMap：
```js
if (entity.stateMap) {
    this._applyStateMap(entity);
}
```

`_applyStateMap` 实现：
```js
_applyStateMap(entity) {
    if (!entity.stateMap) return;
    for (const entry of entity.stateMap) {
        if (this._evaluateCondition(entry.if, this.worldState)) {
            if (entity.currentStateName !== entry.clip) {
                entity.enterState(entry.clip);
            }
            return;
        }
    }
    // 全不命中：回退 initialClip 或 clips 首项
    const fallback = entity._initialClip ?? Object.keys(entity.clips)[0];
    if (fallback && entity.currentStateName !== fallback) {
        entity.enterState(fallback);
    }
}
```

### 3.2 WorldState 变化时重评估

[Scene._onWorldStateChange](file:///e:/se/BlindDuel/scripts/Scene.js#L451-L461) 扩展：在 _pendingSpawns 循环后加一段：
```js
for (const entity of this.entityPool) {
    if (entity.stateMap && !entity.isDisposed) {
        this._applyStateMap(entity);
    }
}
```

仅当新命中 clip 与 currentStateName 不同时才 enterState，避免无谓 texture 切换。

### 3.3 _evaluateCondition 复用

[Scene._evaluateCondition](file:///e:/se/BlindDuel/scripts/Scene.js#L462-L475) 已支持 flag/flagNot/scenario/scenarioMin/scenarioMax/quest，stateMap 的 `if` 直接走它，零新增条件语法。

## 7. 风险与边界

| 风险 | 对策 |
|---|---|
| depthMask plane 与 sprite 不同步移动 | depthMask 挂 root（与 sprite 同父），保证同步 |
| stencil plane 渲染顺序错乱 | renderingGroupId=1 + alphaIndex 取较大值（参考 SceneVisualSystem 用 10000），确保在角色前绘制 |
| stateMap 全不命中时无 clip | 回退 initialClip，再回退 clips 首项（§3.1） |
| prop 被 dispose 后 stateMap 仍被扫 | _onWorldStateChange 检查 `!entity.isDisposed` |
| 同帧多个 stateMap 命中 | 有序数组首个命中即 return，语义明确 |
| PropEntity 进 staticBlockers 后与 StageMask pushbox 重叠 | 美术侧避免祭坛与 StageMask 障碍物重叠；运行时 AABB 重叠无 bug（都挡路） |
