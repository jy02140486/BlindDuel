## 3. 详细变更分析

### 3.1 角色移动系统重构

#### 📋 **CharacterFactory.js** - 默认配置更新
*   **变更内容**: 调整默认角色移动速度参数，实现探索与战斗的速度差异
*   **数值变化**:
    | 参数 | 旧值 | 新值 | 用途 |
    |------|------|------|------|
    | `walkSpeed` | 0.41 | 1.4 | 探索模式移动速度（大幅提升） |
    | `moveSpeed` | - | 0.4 | 战斗模式移动速度（新增） |

#### 📋 **scripts/Enties/CharacterBase.js** - 核心逻辑改造
*   **变更内容**: 
    1. 构造函数新增 `baseMoveSpeed` 和 `activeSpeedMode` 字段
    2. 重构 `getEffectiveWalkSpeed()` → `getEffectiveSpeed()`，支持根据模式选择速度源
*   **关键代码逻辑**:
    ```javascript
    // 构造函数新增
    this.baseMoveSpeed = config.moveSpeed ?? this.baseWalkSpeed;
    this.activeSpeedMode = "walk"; // 'walk' 或 'move'
    
    // 速度选择逻辑
    getEffectiveSpeed() {
        let speed = this.activeSpeedMode === "move"
            ? this.baseMoveSpeed  // 战斗时使用战斗速度
            : this.baseWalkSpeed; // 探索时使用行走速度
        // ... buff 和 tag 加成逻辑保持不变
    }
    ```

#### 📋 **scripts/Systems/Modes/BattleMode.js** - 战斗模式切换
*   **变更内容**: 在进入/退出战斗时自动切换角色的速度模式
*   **新增逻辑**:
    ```javascript
    // 进入战斗：切换到战斗速度模式
    enter(payload) {
        // ... 原有逻辑
        if (combatant) {
            combatant.activeSpeedMode = "move";
        }
    }
    
    // 退出战斗：恢复到行走速度模式
    exit() {
        // ... 原有逻辑
        if (combatant) {
            combatant.activeSpeedMode = "walk";
        }
    }
    ```

#### 📋 **scripts/Systems/Modes/ExploreMode.js** - 探索模式防御性复位
*   **变更内容**: 添加防御性检查，确保进入探索模式时速度状态正确
*   **新增逻辑**:
    ```javascript
    enter() {
        // ... 原有逻辑
        if (character.activeSpeedMode !== "walk") {
            console.warn(`[ExploreMode] character.activeSpeedMode was "${character.activeSpeedMode}"...`);
        }
        character.activeSpeedMode = "walk"; // 兜底重置
    }
    ```

#### 📋 **scripts/Systems/AIKnowledgeRegistry.js** - AI知识库适配
*   **变更内容**: AI知识库读取战斗速度，字段重命名以保持语义一致
*   **变更对比**:
    | 项目 | 旧实现 | 新实现 |
    |------|--------|--------|
    | 值源 | `character.baseWalkSpeed` | `character.baseMoveSpeed ?? character.baseWalkSpeed` |
    | 字段名 | `walkSpeed` | `moveSpeed` |

### 3.2 序章场景配置优化

#### 📋 **Data/SceneDefs/prologue.json** - 场景布局调整

## 4. 影响与风险评估

### ⚠️ **破坏性变更**
*   **无破坏性变更**: 所有变更均通过向后兼容性设计（如 `moveSpeed` 的 fallback 机制）
*   **API变更**: `getEffectiveWalkSpeed()` → `getEffectiveSpeed()` (内部方法，不影响外部调用)

### 🧪 **测试建议**
*   ✅ **验证探索模式速度**: 在序章场景中测试角色移动，确认速度约为1.4
*   ✅ **验证战斗模式速度**: 触发战斗后测试角色移动，确认速度约为0.4
*   ✅ **验证模式切换**: 多次进入/退出战斗，确认速度自动切换无异常
*   ✅ **验证AI知识库**: 检查AI对手的速度数据是否正确反映战斗速度
*   ✅ **验证相机跟随**: 测试序章开场动画，确认相机在200ms后正确跟随同伴
*   ✅ **验证防御性复位**: 尝试跳过 `BattleMode.exit()` 直接进入探索模式，确认警告日志触发
*   ✅ **验证NPC跟随**: 确认同伴跟随行为未受速度拆分影响

### 🎯 **设计亮点**
*   🔄 **模式驱动**: 通过 `activeSpeedMode` 状态字段实现清晰的语义分离
*   🛡️ **防御性编程**: ExploreMode 中添加了状态检查和警告日志
*   📊 **配置驱动**: 速度值集中在 `CharacterFactory` 配置中，便于调优
*   🎨 **场景增强**: 序章场景的条件式行走区域设计支持更复杂的场景演进
