# 26.7.19 prop clip 切换纹理异步重载闪烁修复

## TL;DR
- **现象**：prologue_cs_rabble_flee 序列中 prop_faller 在 land 前后偶发"消失一帧再出现"
- **根因**：`PropEntity._setClip` 切换 clip 时立即 dispose 旧 texture，新建 `BABYLON.Texture` 异步加载，加载期间 spritePlane 无可用纹理 → 渲染为空
- **修复**：构造时并行预加载所有 clip 纹理到 `this._clipTextures` Map，clip 切换仅切引用；`dispose()` 时统一释放

---

## 问题诊断过程

### 1. 初始猜测与日志埋点

最初怀疑几个方向：
- clip 切换瞬间 UV 重算错位
- prop 与 enemy_1 的 alphaIndex 翻转导致渲染顺序闪烁
- y 突变穿越导致 z 穿透
- stencil mask AABB 擦边

在 [PropEntity.js](file:///e:/se/BlindDuel/scripts/Enties/PropEntity.js) 和 [ExploreMode.js](file:///e:/se/BlindDuel/scripts/Systems/Modes/ExploreMode.js) 加了多类 `[JDBG:*]` 日志：
- `[JDBG:CLIP]` clip 切换
- `[JDBG:FRAME]` 帧切换
- `[JDBG:FLICKER]` isVisible / scaling / UV 异常
- `[JDBG:ORDER]` alphaIndex 顺序翻转
- `[JDBG:STENCIL]` stencil 状态翻转
- `[JDBG:Z]` z 位置振荡

### 2. 排除的错误假设

#### 假设 1：alphaIndex 接近 = 闪烁
**错误**：最初检测 `dAlpha <= 10 && dY < 1.5` 就报警。
**修正**：alphaIndex = 1000 - y*100，y 变化时 alphaIndex 必然变化，这是正常的。改为检测真正的顺序翻转（`Math.sign(prev) !== Math.sign(curr)`）+ x 距离判断（屏幕是否真重叠）。

#### 假设 2：主人公离很远也会闪烁
**错误**：检测没考虑 x 距离，hero 在 x=10、prop 在 x=-1.2 也报警。
**修正**：加 `SCREEN_OVERLAP_X = 4` 阈值，x 距离超过直接跳过。

### 3. 真正根因的暴露

跑序列后日志显示：
[JDBG:CLIP] prop_faller: fall → land
[JDBG:TEX_RELOAD] ... clip=land spriteSheetUrl=.../prologue_rabble_flee2.png - texture disposed, new texture loading async...
[JDBG:FLICKER] prop_faller clip=land frame=0 TEX_NOT_READY - texture still loading, prop will be invisible
[JDBG:TEX_RELOAD] prop_faller clip=land texture LOADED
[JDBG:FRAME] prop_faller: 0 → 1 clip=land ...

**关键证据**：
- 每个 clip 用不同 `spriteSheetUrl`（flee1.png / flee2.png / flee3.png）
- 切换瞬间旧 texture 被 dispose，新 texture 异步加载
- 加载期间触发 `TEX_NOT_READY`，spritePlane 渲染为空 → 视觉上 prop 消失一帧
- 没有 `[JDBG:FLICKER] isVisible` 日志 → 不是 spritePlane 被隐藏，是 texture 缺失

---

## 根因

[PropEntity.js](file:///e:/se/BlindDuel/scripts/Enties/PropEntity.js) `_setClip` 的非共享纹理分支：

```js
if (!useShared) {
    if (this.material.diffuseTexture) {
        this.material.diffuseTexture.dispose();  // ← 立即销毁旧纹理
    }
    const tex = new BABYLON.Texture(url, ...);   // ← 异步加载
    this.material.diffuseTexture = tex;           // ← 加载完成前 spritePlane 无纹理
}
```

Babylon `Texture` 构造函数是异步加载，`isReady()` 在加载完成前返回 false。期间 material.diffuseTexture 虽有引用但不可用，渲染时 spritePlane 表现为透明/空。

---

## 修复方案：构造时预加载 + 切换时仅切引用

### 设计要点

| 维度 | 修复前 | 修复后 |
|------|--------|--------|
| clip 切换延迟 | 首次切换需等异步加载 | 零延迟（仅切引用） |
| 纹理创建时机 | 切换时按需创建 | 构造时并行预加载所有 clip 纹理 |
| 纹理销毁时机 | 每次切换销毁旧纹理 | prop dispose 时统一释放 |
| 状态泄漏风险 | 静态缓存跨 prop 难追踪 | 实例级 Map 跟随 prop 生命周期 |

### 改动清单

#### 1. 构造时预加载所有 clip 纹理
`PropEntity.constructor` 新增 `_clipTextures = new Map()` 字段，在 `_buildSpritePlane` 后调用 `_preloadAllTextures()`：

```js
_preloadAllTextures() {
    for (const [clipName, clip] of Object.entries(this.clips)) {
        const url = clip.spriteSheetUrl;
        if (!url) continue;                      // 共享纹理或无纹理跳过
        if (this._clipTextures.has(url)) continue; // 同 url 只加载一次
        const tex = new BABYLON.Texture(url, this.scene, false, false,
            BABYLON.Texture.NEAREST_SAMPLINGMODE);
        tex.hasAlpha = true;
        tex.wrapU = BABYLON.Texture.CLAMP_ADDRESSMODE;
        tex.wrapV = BABYLON.Texture.CLAMP_ADDRESSMODE;
        this._clipTextures.set(url, tex);
    }
}
```

所有 clip 的纹理并行加载，互不阻塞。

#### 2. clip 切换零开销
`_setClip` 非共享分支改为从 Map 取已预加载纹理：

```js
if (!useShared) {
    const tex = this._clipTextures.get(url);
    if (!tex) {
        // 兜底：未预加载则同步创建并存入缓存
        const fallbackTex = new BABYLON.Texture(url, ...);
        this._clipTextures.set(url, fallbackTex);
        this.material.diffuseTexture = fallbackTex;
    } else {
        this.material.diffuseTexture = tex;  // 仅切引用，不创建不销毁
    }
}
```

不 dispose 旧纹理、不 new 新纹理 → 切换瞬间完成，无空窗期。

#### 3. 统一释放
新增 `disposeAllTextures()` 方法，`dispose()` 调用它：

```js
disposeAllTextures() {
    for (const [url, tex] of this._clipTextures) {
        if (tex && tex !== this._sharedTexture) {
            tex.dispose();
        }
    }
    this._clipTextures.clear();
}

dispose() {
    this.disposeAllTextures?.();
    if (this.material?.diffuseTexture) this.material.diffuseTexture.dispose();
    // ... 其余清理
}
```

由 [ExploreMode.js](file:///e:/se/BlindDuel/scripts/Systems/Modes/ExploreMode.js) 的 `disposeProp` handler 触发，sequence 结束时自动清理。

## 经验总结

1. **诊断无规律闪烁，先埋点再下结论**：最初的多个猜测（alphaIndex、stencil、z 穿透）都被日志排除，真正根因是纹理异步加载。如果直接动手改 alphaIndex 逻辑会南辕北辙。

2. **"接近"≠"闪烁"**：检测条件必须基于真正的异常特征（顺序翻转、isVisible 变化、TEX_NOT_READY），而不是数值接近。alphaIndex 接近、y 接近都是正常运动的一部分。

3. **异步资源切换要有过渡策略**：销毁旧资源前确认新资源已 ready，或预加载所有可能用到的资源。这是图形资源管理的通用原则。

4. **诊断日志要分类**：用统一前缀（`[JDBG:*]`）方便过滤，但不同子系统（相机抖动 vs prop 闪烁）应分开命名空间，避免互相干扰。
